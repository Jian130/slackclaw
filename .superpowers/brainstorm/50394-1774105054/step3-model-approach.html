Step 3 Model Setup Alignment

Comparing the realistic ways to bring SlackClaw onboarding step 3 in line with the Figma design for both React and the native macOS app.

Recommended

A. Figma-Exact Curated Step

Show exactly three curated providers in onboarding only, with the same card layout, same step flow, same visual hierarchy, and the same simplified auth panel pattern as Figma.

Pros

  • Matches the requested Figma process
  • Keeps onboarding opinionated and beginner-friendly
  • Lets Config page keep the full provider catalog

Cons

  • Requires step-specific provider curation logic
  • Native and web both need a focused UI rewrite

B. Curated Three + Hidden More

Show the three popular providers first, but keep an escape hatch to reveal the full provider list from onboarding.

Pros

  • More flexible
  • Less risk if provider priorities change

Cons

  • Not the same process as Figma
  • Invites extra complexity into onboarding

C. Keep Dynamic Providers, Restyle Only

Leave the current backend-driven provider list intact and only restyle the screen to look closer to Figma.

Pros

  • Less backend filtering work
  • Lower implementation churn

Cons

  • Still not Figma-exact
  • Onboarding remains more complex than intended

Recommended implementation details

What would change in both React and native
  • Onboarding step 3 shows only three curated providers, in a fixed order.
  • The step uses the Figma card-first flow instead of the current generic provider list.
  • Typography, spacing, card radii, progress header, and auth panel structure are aligned to Figma.
  • Full provider breadth stays available outside onboarding on the normal Configuration screen.
  • Backend still saves a real model entry and performs real auth, but onboarding UI stays simplified.
