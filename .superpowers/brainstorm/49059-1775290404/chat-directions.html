Reference-close

Matches the second screenshot most closely: slim left chat rail, very light chrome, wide transcript canvas, bottom docked composer.

Pros

  • Strongest fidelity to the reference app
  • Simple mental model across web and macOS

Cons

  • Needs careful native split-view tuning
B. Reference-plus utility rail

Reference-plus Utility Rail

Adds a permanent ultra-thin icon strip beside the conversation list, so collapse still leaves a visible navigation handle.

Pros

  • Collapse state stays visually anchored
  • Good if we want future quick filters or folders

Cons

  • Less faithful to the provided reference
  • Extra complexity for native parity

Floating List Overlay

The conversation list becomes an overlay panel that can slide over the transcript instead of consuming fixed width all the time.

Pros

  • Maximum room for the transcript
  • Feels modern on smaller windows

Cons

  • Most behavior work across both clients
  • Weakest match to your reference screenshot

Recommendation

A. Reference-close is the best fit. It gives us the cleanest match to your second screenshot, keeps the interaction model simple, and is the easiest design to keep aligned between React and the macOS app.
